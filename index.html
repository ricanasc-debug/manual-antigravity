PASSO 1

O que é o Antigravity?

Eu não sou apenas um chat de conversa que "cospe" texto. Eu sou um Agente Autônomo de Código com acesso direto ao terminal do seu PC. Tudo o que um Engenheiro de Software Sênior faria com as mãos, eu recrio aqui.

1. Plataformas e Sistemas

Muito além de páginas simples, eu consigo construir coisas gigantes como Plataformas de Vendas, Sistemas de Estoque e até Redes Sociais usando as mesmas ferramentas de ponta que o Google e o Facebook usam hoje em dia.

2. Aplicativos de Celular

Consigo programar aplicativos completos que você pode publicar na lojinha da Apple (App Store) ou Android (Google Play). Eu desenho as telas, configuro a câmera, o GPS e faço a lógica por trás.

3. O Detetive de Bugs

Seus códigos quebraram num projeto obscuro antigo? Eu baixo os logs do terminal, leio infinitas linhas de erro (via ferramenta Read Terminal) e ajusto os arquivos problemáticos um por um para você.

4. Navegador Integrado

Acredite ou não, possuo um Robô-Navegador invisível. Posso abrir o teste de um site recém-criado, preencher um formulário de login sozinho e checar as telas de erro no lugar do seu cliente humano.

5. Pesquisador Autônomo

Não sei a linguagem XPTO? Você manda eu descobrir, eu varro documentações oficiais remotas ou faço pesquisas web independentes até entender as novidades que acabaram de lançar.

6. Manipulador de Backend

Programo seus servidores, manipulo tabelas de Banco de Dados via "MCP", crio APIs do zero com Node/Python e executo migrações complexas gerando e rodando os comandos de terminal.

PASSO 3

Habilidades Especiais (Skills)

Skills são como "cartuchos de videogame" que você espeta em mim. São pastas cheias de conhecimento fechadinho (como regras e scripts de negócio) e que eu passo a ler e respeitar como se fossem a bíblia.

Como preparar uma Skill para eu usar:

1

Descubra a pasta: Normalmente as Skills ficam salvas num diretório próprio do agente (Pode me pedir para verificar essa estrutura de pastas onde estamos salvando).

2

Crie o arquivo principal: Na pasta da Skill, crie um documento chamado SKILL.md. Lá dentro, você escreve em linguagem humana tudo aquilo que "Eu nunca posso esquecer ao rodar este tipo de projeto".

3

Rechone a Skill de assets: Se a Skill exige coisas (Templates, arquivos), crie uma subpasta /resources ali junto.

4

Como usar: Sempre que quiser me invocar naquela Skill, é só falar comigo na conversa normal: "Vá no seu arquivo de Skills e siga as instruções do SKILL X". Eu vou ler rapidamente e seguir à risca!

PASSO 4

Conectando Ferramentas Externas: MCP

MCP significa Model Context Protocol. Parece palavrão de engenheiro de foguetes, mas na verdade é só um "Cabo Universal" inventado para eu me plugar aos seus programas externos!

Banco de Dados

Podemos configurar um MCP onde eu ganho acesso direto para vasculhar o seu banco PostgreSQL remotamente e sugerir comandos SQL ou achar erros sozinhos nos seus dados.

GitHub Server

Se você plugar um MCP do Github, eu consigo ler Repositórios do Github (issues, códigos) pela nuvem, analisar erros do sistema alheio, entre outros.

A Receita Absoluta do MCP (JSON de Configuração)

Aqui está o passo a passo de como plugar um banco de dados real na minha inteligência artificial usando MCP via Node.js (npx):

1

Encontre o Servidor MCP: A comunidade já programou conectores pra tudo. Digamos que você queira que eu leia um banco de dados chamado SQLite. Você usará o pacote: @modelcontextprotocol/server-sqlite.

2

Configure O "Mestre": Para eu ganhar o superpoder, você precisa achar o arquivo de configuração do sistema que me roda (como por exemplo o claude_desktop_config.json ou as Configurações do Antigravity) e declarar a invocação usando JSON puro:

mcp_config.json
{
  "mcpServers": {
    "sqlite_local": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-sqlite",
        "C:/caminho/do/seu/bancodedados.db"
      ]
    }
  }
}
3

A Mágica do Reinício: Salve o arquivo JSON e reinicie a minha interface (esteja fechando e abrindo o app, ou reiniciando o servidor Antigravity). Quando eu acordar, olharei esse JSON e abrirei túneis de dados invisíveis. Sem que você veja, irei ler list_resources e list_tools daquele servidor.

4

Usando o Poder: Acabou a teoria! Agora que a chave está ligada, você fala na nossa janela de Chat normal: "Acesse o meu banco de dados de clientes usando o MCP e me diga quanto dinheiro a empresa lucrou". Eu usarei as ferramentas recém descobertas sozinho pra fazer a varredura SQL real na sua máquina!

PASSO 5

Como criar Sites e Landing Pages 🎨

Entenda os 3 níveis de complexidade estética de Sites que você pode construir usando Inteligência Artificial. Suba de nível conforme a sua ambição.

1. Site Simples (Rápido e Funcional)

O feijão com arroz perfeito. Um site liso estruturado para apresentar o seu pequeno negócio logo de cara. Foca puramente na entrega veloz da informação básica.

1

A Receita de Entrada: "Crie uma Landing Page simples para o meu consultório unificada em 1 só arquivo. Use fundo branco claro, coloque um título escuro grande, uma área vazia para eu tacar minha foto depois, um parágrafo 'Quem sou eu', e lá no fundo um botão verde gigante chamando pro meu Whatsapp."

2. Site Médio (Moderno e Dinâmico)

Entrando nas tendências de UI modernas. Aqui nós paramos de empilhar texto e incluímos formas agradáveis, caixas e caixinhas da moda.

2

A Receita Elegante: "Suba o nível visual daquela Landing Page. Quero que a área dos 'Meus Serviços' se torne uma Grade (Bento Grid, como de Fintechs) e que minhas perguntas da seção de Dúvidas (FAQ) se escondam. Transforme elas num Acordeão: caixas que abrem e fecham perfeitamente quando o cliente clica. Traga a fonte Roboto direto do banco Google Fonts pra ficar elegante."

3. Estilo Apple Supremo 🍎 (A Receita Mastigada)

Quer fazer aquele site nível Curso Profissional (com visual premiado) sem enlouquecer? O segredo é fazer em etapas. Nunca tente fazer o site todo de uma vez só! Siga meus 5 passos:

1º

Ache uma Cópia (Sem vergonha!)

Ninguém tira uma ideia do nada. Vá no site Pinterest.com e pesquise: "Dark Mode Landing Page". Ache a imagem de um site que você acha chique e tire um "Print". Iremos copiar o estilo bruto dele.

2º

Faça o Desenho (No Papel!)

Pegue um papel de caderno e uma caneta. Rascunhe: "A foto principal fica nesse quadrado em cima, os 3 serviços ficam embaixo." Tire uma foto pelo celular e me mande no chat junto com a inspiração.

3º

Eu construo a Casa Seca

Mande para mim:
"AG, crie o HTML desse site que eu desenhei com as cores brutas do modelo (Fundo escuro, caixas arredondadas)."
Eu levantarei as paredes. Vai nascer, mas ainda estará durão.

4º

Eu Injeto a Magia da Apple

Com a casa erguida, você ordena:
"A estrutura ficou ótima. Agora injete a animação fluída com GSAP. Faça as caixas brotarem quando eu rolar a tela do site pra baixo."
Aqui ele ganha vida!

5º

A Cereja Oculta (O 3D Transparente)

GAVETA DE MÍDIAS 🖼️

Fotos, Vídeos e Estética (A magia visual)

Houve um tempo em que as pessoas sofriam meses pra fazer uma foto aparecer no site ou rodar aquele vídeo luxuoso no fundo da tela. Acabou. O segredo de injetar mídia local é ridicularmente fácil, mas exigente de regras:

1

A Gaveta Obrigatória (`assets`): O erro matador que você descobriu antes é tentar usar imagens lá da pasta de "Downloads" do Windows. Quando eu for subir seu site pro mundo na hospedagem, o computador gringo da nuvem não terá acesso à sua pasta Downloads. Por isso, SEMPRE crie uma pastinha de nome assets ou img ao lado do seu arquivo index.html. Pegue fotos e vídeos de Arquitetura e jogue tudo ali dentro.

2

Nomes Limpos e Sem Espaços: Computadores gringos odeiam espaço e ponto bizarro. Se o arquivo chamar Foto Final (2) Cóópia!!!.jpeg, o site vai quebrar. Dê botão direito e renomeie o arquivo de forma super limpa, com tudo em letras minúsculas: fachada-loja.jpg ou logo-roxa.png. E se a imagem for feia, é simples, diga para Mim: "Antigravity, joguei uma imagem no meu index, mas faça ela ficar maravilhosa e redonda, adicionando um efeito Cover CSS!"

3

No Cinema (Vídeos Autoplay): O luxo supremo! Se você encontrar um vídeo sem som bem limpo e quiser como tela de fundo inteira do site, só avise e ordene: "Antigravity, pegue este vídeo chamado 'fundo-home.mp4' na pasta 'assets', e coloque ele rodando em Loop e Silenciado (Muted) por trás de todo o letreiro principal esmaecido da minha Landing Page para dar o efeito Cinema Absoluto".

BÔNUS AVANÇADO

Sistemas & Autenticação (Acesso Restrito) 🔒

Uma Landing Page é apenas uma vitrine pública. Mas e se você quiser criar um aplicativo fechado, que as pessoas precisem digitar Senha para entrar num Banco de Dados próprio? Bem-vindo ao mundo dos Sistemas.

1. O Frontend e o Backend (As duas metades)

Para criar um sistema verdadeiro (Dashboard Seguro), a nossa criação sempre se dividirá em duas metades obrigatórias:

1

A Rosto Visível (Frontend): Vamos criar a maquiagem da tela do Site (Formulário do Login). Nenhuma senha é guardada aqui, é só botões e visual cosmético.

2

O Cofre Invisível (Backend): Em seguida, crio a programação profunda com meu código Mestre (Node.js ou Python) que pega a senha digitada nesse rosto, joga pro Banco de Dados, averígua a matemática pesada trancada, e destranca a porta.

3

O Comando para Mim: "AG, crie um Sistema de Estoque. Inicie pelo 'Backend'. Programe as rotas invisíveis e conecte a um Banco de Dados Local (SQLite). Só quando estiver perfeito nós faremos o Frontend com as cores para ligar os botões."

2. BaaS (Bancos Prontos na Nuvem - Firebase / Supabase)

Se você não quer ter dor de cabeça configurando Bancos de Dados difíceis na sua máquina, usamos uma nuvem terceirizada chamada "BaaS". É como se fosse um "Google Drive glorificado" apenas para estocar senhas de sistemas e tabelas com mega segurança, sem você precisar sujar as mãos.

1

Como Funciona: Você vai na internet, digita no Google Supabase.com ou Firebase.com e faz uma conta gratuita e limpa. Eles te dão uma "Chave Secreta API".

2

A Mágica no Chat: "AG, construa minha tela de Login e a conecte no BaaS Supabase. A minha Chave Secreta do Supabase é '123xpto'. A partir de agora, jogue a conferência e o salvamento de clientes lá no servidor mestre deles!"

3. Hospedagem Focada em Sistemas (Vercel ou Render)

Hospedar sua "página estática de publicidade gratuita" no GitHub (Passo 6) funciona lindo. Porém um Sistema Pesado com Robô Backend não pode ficar numa página gratuita do GitHub. Ele precisa de maquinários de Servidor vivo processando as contas matemáticas.

Cuidado com a Hostgator / Hostinger!

Não compre o plano básico dessas empresas para colocar Sistemas/Logins. Elas são como "Casas de Residência" (ótimas para hospedar sites estáticos e WordPress simples). Nosso sistema é uma "Fábrica Industrial" que exige servidores tipo Cloud/Node pesados, que nelas vai custar caro e te dar dor de cabeça imensa pra instalar.

1

A Escolha Premium Automática: Recomendamos fugir do convencional e assinar a plataforma especializada em "Fábricas Modernas" grátis/paga chamada Vercel.com ou Render.com.

2

Publicando: Elas são geniais. Você faz o seu passo a passo do GitHub (Passo 6 regular), entra num painel da Vercel.com, clica pra associar o seu GitHub nela e pronto! Ela instala o Node.js como mágica instantânea e mantém seu Login online o ano inteiro.

A CEREJA DO BOLO 🍒

O CRM Perfeito (Sistema de Gestão) 🎯

Pequenos negócios não precisam pagar caro pelo Hubspot. Nós podemos comandar o Antigravity para criar um "Caderninho Digital Supremo" personalizado para as suas exatas necessidades de vendas. Veja o plano de voo.

O Plano de Orquestração do CRM Mestre

Você nunca pede pra IA "criar um CRM inteiro hoje". Sistemas são como montar carros; começamos pelo chassi e terminamos na lataria.

1

A Base (O Banco de Dados): Antes de ver cores, crie o cofre. Comando: "Antigravity, crie as tabelas do meu CRM de Arquitetura no servidor (Supabase). Preciso de 1 Tabela para [Contatos do Cliente], 1 tabela para [Serviços Orçados], e 1 tabela que marque a [Fase da Venda: Prospecto, Negociando, Fechado]."

2

O Quadro Kanban (A Lataria): Essa é a cara clássica do CRM Moderno. Comando de Frontend: "Agora crie o Painel protegido conectando com aquelas tabelas. Quero o visual no formato 'Quadro Kanban' (igual ao Trello). Onde eu possa pegar o card do cliente fulano que 'Ligou hoje' com o Mouse e soltar e arrastar ele pra coluna 'Assinou Contrato'."

3

A Automação Mágica (O Piloto Automático): Conecte tecnologias! Comando: "Antigravity, adicione a biblioteca de disparo do WhatsApp. Faça uma ligação (Webhook) com o Frontend. Todas as vezes que eu pegar o card de um Cliente e arrastar para a coluna 'Fechado', o meu servidor tem que enviar magicamente uma mensagem automática pro celular dele dizendo: Obrigado por fechar com nossa empresa!"

4

O Salto Evolutivo (Conectando ao Dashboard): Um CRM puro é só o setor operacional. Para os diretores verem o lucro subindo de verdade, ele precisa virar um Painel. Diga: "Antigravity, conecte minha tabela do Kanban à tela gráfica do meu Dashboard para o Chefe ver a pizza piscar em Tempo Real." (Veja mais na aba O Dashboard Moderno logo abaixo).

Dica da Tiazinha: Comece pequeno! Crie um CRM só com nome e telefone. Peça pra subir isso e veja se roda. Quando a rodinha de treino rodar, aí você manda o sistema incluir automação de WhatsApp, disparo de e-mail e gráficos de comissão!

DASHBOARDS LUXUOSOS 📊

Como Criar um Dashboard Moderno e Bonito

Esqueça os "Painéis Administrativos" cinzas com tabelas chatas dos anos 2000 que dão sono. Hoje, o sucesso de um sistema mora na estética "Premium" que faz o funcionário querer usar a ferramenta o dia todo. Eis as 3 regras de ouro do Design:

1

A Grade "Bento Box" (Estilo Apple): Todo Painel chique não usa seções separadas por "chão vazio". Eles constroem uma grande tela de fundo escura, e colam as informações flutuando em Caixas com cantos arrendondados suaves, como aqueles "Widgets" modernos no seu iPhone ou nas "Marmitas Japonesas Bento".

2

Cores de Alto Contraste (Dark/Glass): Exija um fundo Preto fosco ou Roxo super denso (Dark Mode). Em vez de bordas brancas duras, peça para Aplicar "Glassmorphism" nas caixas (elas ficam como vidro fosco que deixa vazar uma fumaça de neon colorido do que está por trás da tela).

3

Gráficos Livres (Chart.js / ApexCharts): Tabelas escritas não têm luxo. Toda meta, avanço e finança tem que subir sozinha usando as poderosas "Bibliotecas de Graficos em Javascript". Quando a IA injeta Chart.js, seus gráficos de pizza ganham vida girando na hora que você acessa.

O Prompt Perfeito da Tiazinha p/ um Dashboard Misto:

"Antigravity, aja como um UI Designer Sr da Apple. Construa um Backend de Dashboard de Vendas (Painel de Gestão). Aplique tema puramente Dark Mode, usando a arquitetura 'Bento Grid' onde todos os meus números e cartões flutuem no fundo com cantos super macios. E crucial: adicione a biblioteca ApexCharts pra que os meus gráficos de Faturamento da semana subam em verde neon fazendo uma animação elástica e gostosa quando a tela for ligada no botão!"

O Casamento Perfeito: Integrando CRM ao Dashboard 💍

O segredo de um projeto milionário é não tratar sistemas como ilhas separadas. O seu CRM (O Caderninho de Vendas) deve ser o Cérebro Injetável do seu Dashboard (A Apresentação).

1

A Sincronização Matemática: Em vez do Gráfico de Pizza no seu dashboard apresentar números "falsos" (digitados na mão por um funcionário), você pede pra nossa IA escrever um "Cabo Vísivel" do seu Banco de Dados (Supabase). Quando alguém arrastar o cartão de um paciente para a coluna "Pagou" lá no CRM Kanban de Vendas, o código matemático no fundo da nuvem corre até o seu Dashboard e faz a linha do Gráfico do Faturamento do mês subir sozinha e piscar em verde.

O Comando de Integração Máxima:

"AG, eu quero casar meu fluxo. Programe os arquivos do meu Dashboard Luxuoso para que os Gráficos do 'ApexCharts' leiam APENAS a tabela mestra de 'Clientes Pagantes' do meu Banco de Dados lá no Supabase. Se o vendedor arrastar uma venda pro painel do CRM Kanban hoje, o número grande flutuante na tela de Dashboard principal do Chefe da empresa deve atualizar em tempo real em menos de 1 segundo sem precisar ficar apertando F5 pra atualizar a página!"

BOTÃO DE EMERGÊNCIA 🚨

S.O.S - O Código Quebrou, e agora?

A regra absoluta número 1 dos engenheiros chefes é: Sistemas SEMPRE vão quebrar e dar conflitos algum dia. Às vezes o pacote oficial da gringa atualizou e o Antigravity sem saber tentou injetar um código antigo, aí a TELA PRETA DO TERMINAL VOMITA MIL LETRAS VERMELHAS BIZARRAS.

Quando isso saltar na sua cara, NÃO CONGELE O CORPO ACHANDO QUE FEZ ALGO ERRADO.

A Tática do Desespero (Buscando A Cura) 🔥

Aquela sopa de letrinhas caótica vermelha cheia de `ERR!` não é ameaça. É, na verdade, o próprio Computador tentando desesperadamente nos contar ONDE, EM QUE LINHA e POR QUE não deu certo.

1

A Copiada de Ouro: Pegue seu mouse, vá no Terminal (ou no navegador) que travou, selecione ABSOLUTAMENTE TUDO que estiver rabiscado de vermelho com a palavra Error, do primeiro ao último ponto. Copie (CTRL+C) sem dó nem piedade.

2

O Comando Mágico Curador: Volte e jogue toda aquela bagunça incompreensível de "Traceback/ReferenceError" em mim (No Chat Antigravity) e mande este comando de exorcismo:

"AG, socorro! Eu executei isso e deu esse erro horroroso gigante abaixo. Leia cada linha dele com calma. Me explique qual pacote ou arquivo foi o culpado como se eu fosse criança, e CORRIJA meus arquivos magicamente pra fazer a tela funcionar! Segue o laudo: [COLE O ERRO AQUI]"

PASSO 6

Publicando na Internet (Hospedagem) 🌍

O site está pronto! Como tirar ele do seu computador e colocar na internet para seus clientes acessarem? Veja o passo a passo detalhado abaixo.

O Guia Definitivo: Qual Hospedagem Escolher? ⚖️

1. Landing Page Simples / Portfólio (Sem o seu próprio ".com.br")
🟢 Custo: 100% GRÁTIS (Pra Vida Toda)
Onde hospedar: GitHub Pages.
A Vantagem: O código fica trancado online num cofre e os servidores gigantes do projeto open-source rolam seu site tão rápido que ele literalmente tem zero chance de cair por lotação. (Siga a Opção 1 abaixo da tela)
2. Site Vitrine Padrão da Empresa (Com "www.EmpresaDoSeuNegocio.com.br")
🟡 Custo: PAGO (Uma anuidade ou mensalidade barateza de praxe)
Onde hospedar: Hostinger ou Hostgator.
A Vantagem: Elas conectam o seu endereço pago Brasileiro .com.br rapidinho com o servidor de fachada delas. Compre APENAS os planos Base (Ex: "Hospedagem M" da Hostgator). É suficiente pra sites estáticos. (Siga a Opção 2 abaixo da tela)
A Dúvida de Ouro: Preciso pagar para o site rodar?

A maioria das Landing Pages ou "Vitrines de Serviço" que fazemos são Sites Estáticos (apenas fotos, textos e botões enviando pro WhatsApp). Eles nunca vão precisar de hospedagem paga mensais. Eles moram perfeitamente de graça no GitHub.

Você só precisará tirar a carteira do bolso em 2 únicas situações:
- 1º: O E-mail Chique. Se a empresa encomendar um e-mail que termine com contato@nomedaloja.com.br, o GitHub não cria e-mail. Aí você compra aquelas Hospedagens Tradicionais (da Opção 2) que servem justamente para dar essas caixas postais oficiais.
- 2º: Lojas e Sistemas Pesados. Se no futuro fizermos uma "Loja Virtual Oficial" que gerencia carrinho e desconta saldo de cartão nela mesma, ou tivermos que instalar o pesadão do *WordPress*, a casa de graça fica apertada.

3. Máquinas Vivas de Banco de Dados (CRM, SaaS, Dashboards com Sistema ou Senha)
🔵 Custo: GRÁTIS (No começo), pagando apenas quando você tiver centenas de clientes usando.
Onde hospedar: Vercel.com ou Render.com.
A Vantagem: Nem todo dinheiro do mundo faz um código "Backend de Node/Python" pesado rodar bonito e limpinho nas Hospedagens Baratas da Opção 2 sem você ter dor de cabeça técnica monumental. Opção 1A: O Piloto Automático do GitHub (Pela I.A.)

O jeito mais rápido. Se lembre que nós já conectamos nossos fios aqui: esqueça aplicativos desajeitados; peça para o cérebro enviar por linha de comando invisível!

1

Crie o Cofre Seco no GitHub.com: Faça o login lá no site oficial e clique no botão verde "+" no canto superior direito para criar um "New Repository". Dê o nome do seu projeto ali e crie. Ficará uma tela cheia de códigos, ignore tudo.

2

O Comando Mestre (Eu trabalho por você): Vá para a nossa aba do próprio chat e mande a ordem suprema: "Antigravity, faça o Push (Envie) os códigos que fizemos hoje para o meu repositório 'xyz' no GitHub que acabei de criar". Eu vou abrir o terminal sozinho e rodar todas as linhas difíceis direto por conexão sem mexer na sua mão!

Opção 1B: O Caminho Visual Tradicional (GitHub Desktop)

Você é mais visual e prefere arrastar pastas com o mouse ao invés de pedir pra IA? Use o aplicativo do computador!

1

Crie a conta no GitHub.com: Faça o mesmo passo anterior, vá no site e clique no "+" para criar um "New Repository".

2

Use o GitHub Desktop: Baixe o aplicativo verdinho "GitHub Desktop" no PC e faça o seu login. Clique em File > Add local repository e selecione a própria pasta do seu site na sua máquina.

3

Publique (Suba para a Nuvem): Após adicionar a pasta, digite qualquer nome na 'Summary' no canto esquerdo e aperte `Commit`. Depois clique no botão azul gigante Publish repository no alto da tela do app. ATENÇÃO: Para o site pegar, desmarque a caixinha "Keep this code private" na hora que a janela abrir!

Como eu coloco isso No Ar? (Aplica-se nas Opções 1A e 1B)

🔥

Ligue a Antena Final (A cereja do bolo): Depois que os arquivos chegarem no site (seja pela I.A ou pelo Desktop), abra o GitHub.com. Entre no seu repositório, clique na engrenagem Settings no menu superior, e depois em Pages na barra lateral esquerda. No meio da tela, troque "Branch" de "None" para a aba main (ou master) e clique em Save. É só aguardar 2 minutos e a tela vai te dar o link verde oficial do seu site no ar!

Opção 2: Hostgator / Hostinger (Plano `.com.br` Profissional)

Ideal quando você compra o seu domínio próprio na internet (ex: www.suaempresa.com.br) e contrata um provedor comercial robusto.

1

Empacote seu site (O Famoso ZIP): Na pasta original do seu computador, selecione TODOS os arquivos soltos (o `index.html`, pastas de fotos, `style.css`), clique com botão direito e escolha a opção do Windows Compactar para arquivo ZIP. Você terá apenas 1 arquivinho aglomerando tudo.

2

Acesse o Painel da Hospedagem: Faça seu Login, acesse a conta da Hostgator (lá dentro eles usam um sistema clássico chamado cPanel) ou Hostinger (nomeado de hPanel).

3

Abra o "Gerenciador de Arquivos": Localize nas abas e botões do provedor o "Gerenciador de Arquivos" (File Manager). É aqui que fica o HD/Computador virtual deles.

4

Ache o coração da internet `public_html`: Clique nesta pasta. Tudo que está dentro dessa exata pasta public_html é o que o mundo vê. Você pode e deve APAGAR o que já estiver lá (como "default.php" ou propagandas da provedora).

5

Faça o Upload do Zip: Clique no botão de Upload / Carregar Arquivos no topo da tela e suba o seu pacote .zip.

6

Extraia o sucesso: Por fim, clique com botão direito no seu arquivo `.zip` já subido e clique em Extract (Extrair). Seus arquivos originais saltarão para fora espalhados na pasta public_html. (Verifique se o "index.html" não ficou escondido dentro de alguma subpasta). Se o index está cru na raiz, entre no seu site no celular e comemore!

PARA QUEM NUNCA PROGRAMOU

A Anatomia de um Site (A Trindade da Web) 🏠

Eu construo quase tudo por você, mas é fundamental que você saiba o que significam aqueles 3 arquivos misteriosos que sempre criamos na nossa pasta (index.html, style.css e script.js). Pense em um site como se fosse a construção de uma Casa!

1

O Arquivo HTML (Os Tijolos e o Esqueleto):
Se tivéssemos apenas o arquivo index.html, a sua página de internet seria uma parede feia no tijolo baiano. O HTML não dá cor, ele apenas dita "AQUI TEM UM TÍTULO", "AQUI TEM UMA FOTO", "AQUI TEM UM BOTÃO". Ele é o conteúdo puro em preto e branco.

2

O Arquivo CSS (A Tinta e o Decorador):
Aqui a mágica do design acontece! O style.css é o nosso reboco bem feito e o porcelanato italiano. Ele pega aqueles tijolos duros do HTML e diz: "Pinte esse botão de Rosa Neon, arredonde os cantos dele e mova aquela foto para a direita". É ele quem deixa as coisas visualmente luxuosas, ou a sua vitrine passaria vergonha.

3

O Arquivo Javascript (A Eletricidade e os Motores):
Uma casa pintada não serve para muito se as luzes não acendem quando você aperta a tomada, concorda?! O arquivo script.js entra trazendo eletricidade. É ele quem dá a lógica. Ele diz: "Se o cliente clicar neste botão chique, quero que uma mensagem de sucesso salte na tela e envie os dados dele para a nuvem!". É a pura interatividade.

Abrindo os arquivos sem medo

Como a senhora é a Arquiteta Chefe do projeto, nunca tenha pânico de abrir o código e fuçar. Se quiser mudar uma letrinha rápida no Letreiro do site sem sequer gastar energia conversando comigo, basta abrir o index.html em um bloco de notas, dar um CTRL+F (para achar a palavra exata que quer trocar), apagar, digitar por cima e Salvar o arquivo no menu!

O GUIA DO CONTEÚDO

Blog: Mitos e Verdades (Qual escolher?) 📝

Todo mundo acha que Blog de Notícias precisa custar caro na mão de agências, mas as regras mudaram. Veja a diferença entre a estrada antiga e a nova.

1

A Estrada Antiga (O famoso "WordPress"):
Esse é o jeito pesado que as empresas usavam no passado (e muitos ainda usam). Ele exige um "Estoque Cego" fortíssimo (Backend). Você não escreve na tela de frente, você abre um painel administrativo com senha no fundo da loja e escreve por lá.
O grande mito: "Preciso do WordPress pra ter um blog". Mentira. O problema é que o WordPress não roda no GitHub grátis. Você obrigatoriamente terá que pagar a mensalidade da Hospedagem Paga pelo resto da vida.

2

A Estrada Moderna (O Blog Estático):
A febre atual. Mais rápido e à prova de hackers (ninguém consegue invadir se não existe senha pra invadir).
A verdade genial: O Blog não passa de um monte de arquivos de texto (igualzinho ao arquivo deste Manual). Sendo assim, ele **roda perfeito e de graça** naquelas hospedagens limpas do GitHub Pages das lições Anteriores.

O Truque de Mestre (Blog usando Notion)

Se a senhora (ou o dono da loja do projeto) preferir escrever os artigos no conforto do celular de forma fácil pra postar no site, nós criamos o "**Blog Notion**".

O Notion é um aplicativo genial e gratuito (como se fosse um bloco de anotações misturado com Word). A nossa inteligência Cria um "fio invisível" entre o seu aplicativo do Notion e o Site da Vitrine. A senhora escreve no caderno branco pelo App no sofá usando o teclado do celular, o nosso código "puxa" o texto de lá na surdina e publica automaticamente, lindo e decorado nas paredes de vidro da sua loja virtual (Sem gastar R$ 1 de hospedagem super faturada).

E os Vídeos ou Áudios pesados?

Os textos e as fotos nós puxamos sem problemas do aplicativo. Mas NUNCA devemos salvar arquivos de vídeo gordos (como um arquivo `.mp4` de 500MB) lá (isso derrubaria a perfeição e a velocidade do seu site). A regra de ouro é: **Suba o seu vídeo num canal do YouTube oculto**, tire o link de lá e só cole no texto. O gigantesco Google gasta a energia de servidor dele rodando o vídeo para o cliente, enquanto o seu blog continua voando liso pela internet!

O AdSense e a Monetização (Google)

O Google não faz distinção se o texto veio de um Notion gratuito ou de um WordPress caríssimo. Nós (a Inteligência) simplesmente injetamos seu Código JavaScript Oficial do AdSense na estrutura do Mural estático e os anúncios vão pipocar! O maior benefício? Como nosso sistema do GitHub é ridicularmente veloz, o algoritmo de *SEO do Google* se apaixona e **tende a jogar seus artigos para a primeira página** de busca muito mais vezes!

A ARTE DE APARECER

SEO: A Magia de Rankear no Google 🔍

Dizem que colocar seu site no topo do Google (SEO) é um bicho-papão cheio de segredos. A verdade é que existem apenas 4 regras fundamentais. E a melhor parte: O Antigravity faz 3 delas sozinho para você.

1. A Pista Expressa (Carregamento Ultrarápido)
O Google odeia sites que ficam com a tela branca carregando. Como todas as nossas Vitrines e Blogs estruturados aqui são baseados em "Hospedagens Estáticas e Limpas" via GitHub, elas abrem instantaneamente e isso nos dá pontos valiosíssimos de Rankeamento.

2. Os Crachás Invisíveis (Meta Tags e Imagens)
O robô do Google não tem olhos para ver fotos. Quando eu construo a página para você na surdina, eu escondo etiquetas pelo código dizendo ao Google: "Essa foto aqui é um Galpão Metálico", "Este é o Letreiro Oficial". Isso é OBRIGATÓRIO constar no código para ele exibir a sua foto na aba Imagens.

3. A Placa Mestra (A Regra do H1)
Nós só podemos ter 1 ÚNICO TÍTULO PRINCIPAL (que chamamos de Título H1) por página. Eu programo o esqueleto HTML obedecendo essa lei cega do Google, organizando todos os outros blocos em subcategorias perfeitas para o robô de leitura deles não se confundir.

4. O Seu Único Trabalho: Palavras de Sofrimento

Essa é a 4ª e última regra, e a responsabilidade é toda sua, Arquiteta Chefe! Muitas empresas adoram escrever no topo do site: *"Excelência e Dinamismo Inovador"*. Nenhum cliente do planeta acorda de deprimido e digita "dinamismo inovador" na pesquisa do Google!

O truque supremo é: Escreva apenas as **"Palavras de Sofrimento"**. Quando me pedir pra criar a vitrine, mande eu pregar nos títulos as exatas palavras que o seu cliente digita na hora do desespero: "Empresa que conserta Telhados Vazando em São Paulo SP". Faça a sua parte e deixe todo o sacrifício numérico pro seu Programador IA!

BÔNUS

Dicionário da Tiazinha (Tech Analógico) 📼

Esqueça o jargão complicado dos programadores mais novos. Aqui traduzimos os nomes feios da tecnologia para as coisas do dia a dia (e dos anos 80) que você já domina muito bem!

Frontend (A Vitrine)

O Frontend é a "vitrine da loja de shopping". Tudo que o seu cliente vê, clica e acha bonito no site é o Frontend. É a maquiagem, a arquitetura visual e a cortina do palco.

Backend (O Estoque)

O Backend é o departamento de "Estoque" (ou a Cozinha do Restaurante). O cliente não vê a cozinha, mas é lá que ficam as receitas, as senhas, os ingredientes e os cozinheiros preparando o dado puxado do forno.

Bug (A Fita Mastigada)

Quando dizem que "deu bug", significa que rolou um defeito. É como se a fita cassete tivesse mastigado no videocassete ou a fita da máquina de escrever tivesse enroscado. Algo parou e precisa de reparo técnico.

Deploy (A Inauguração)

Fazer "Deploy" é soltar os fogos da inauguração. É tirar aquele site que só funcionava em modo rascunho na nossa máquina e jogar ele pra rodar na antena da rádio pro mundo inteiro visitar livremente.

Cloud / Servidor Node (A Fábrica Longe)

A "Nuvem" não voa no céu. Ela é apenas um Supercomputador trancado em um galpão seguro de empresas como Vercel ou Google. E o poderoso **"Node" (ou Node.js)** é o motor de esteira gigante que fica instalado lá rodando 24h por dia para aguentar receber o login pesado de todo mundo de uma vez só.

SaaS (O Sistema de Aluguel)

Antigamente você ia nas Lojas Americanas, comprava o CD-ROM do Microsoft Office e instalava na máquina pagando uma vez só. SaaS (Software como Serviço) é o contrário: você não baixa nada, acessa o botão de Login num site bonito e paga uma "mensalidade de aluguel" para poder usar a ferramenta. Exemplos famosos de SaaS: Netflix (mensalidade de filmes), Canva (mensalidade de design), Spotify ou Sistemas de Clínicas.

NPM / Bibliotecas (O Lego)

Imagine colar figurinhas. "Bibliotecas" ou "NPM" são pacotes de código que já vêm prontos. Em vez de eu, da Inteligência, precisar inventar e programar uma "roda redonda" do zero todo dia, a gente pega a roda pronta do Lego que alguém fez na comunidade de graça e encaixa ali!

CRM (A Agenda Milagrosa)

Um "CRM" hoje em dia é disparado o tipo mais famoso de sistema SaaS. Lembra do velho caderninho onde o dono da loja anotava o nome do freguês, quem pagou e quem sumiu? O CRM é o caderninho supremo. É um SaaS que salva o histórico inteiro do cliente, diz em que etapa ele parou na compra e ajuda sua equipe a vender. Hubspot e RD Station são grandes exemplos.

GSAP (Efeitos Especiais)

Pense nele como o nosso "Diretor de Hollywood" das animações e magias do luxo visual. O GSAP é um pacote (ou Ferramenta de Animação veloz) que eu injeto no site para que as imagens e os textos deslizem perfeitamente na tela do cliente como se fosse um Cinema; tudo sem travar o processador dele.

Supabase / Firebase (O Cofre Alugado)

No lugar de você comprar um cofre de metal pesado, tentar instalar na sua casa e correr o risco de pegar fogo, você aluga uma gaveta no "Banco Suíço". O Supabase e o Firebase são empresas onde a gente cria uma conta de graça, eles te dão o cofre pronto na nuvem deles, e nossos Códigos (O Backend) enviam os dados pra salvar lá dentro em segurança máxima.

Vibe Coding (O Design Falado)

É uma gíria moderninha pra quando você não encosta em programação pesada. Você apenas "dita" ou esboça com ferramentas (como o Google Stitch) seguindo sua aura ("Vibe") de criatividade do momento, e a Inteligência pinta as telas de vitrine perfeitamente baseada unicamente no que saiu da sua fala.
